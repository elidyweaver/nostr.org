Notes and other stuff transmitted by relays

Nostr is a distributed social network: users publish notes to multiple servers.

Read the Protocol

Interact with the world and find your niche communities

Each server is ran by a different person and each has its own rules. You only interact with the servers you desire, or you can just fetch the notes that you want specifically from any server and ignore the rest.

Signed at the core

Instead of using insecure passwords, all messages are signed using cryptography. This way you can verify messages & user authenticity without relying on a centralized component.

FAQs

What's with the Ostrich?

The Ostrich is the beloved mascot of Nostr users.

Why are DM's not considered secure?

DM's on Nostr using the NIP-04 are encrypted, but only it's contents. This means that people can discover which DM's are being sent to who. This is different from secure chat applications like signal where all communications are encrypted.

What are NIPs?

NIPs stand for Nostr Implementation Possibilities. They are standards for implementing required or optional features through the Nostr protocol.

Why are browser clients considered unsafe?

This is a valid concern, but web clients are safe as long as they ensure to code against regular attacks such as Cross Site Scripting. For web developers this can be easily mitigated with a strict content safety policy. Regardless of what client you are using, you should always make sure you trust the author of the software.
